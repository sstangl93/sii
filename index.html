Build

Architecture and Envisioning

Architectural Design Process (ADP)
  • Define project scope
  • Discover/gather the architectural drivers (quality attributes)
  • Create notional architecture
  • Review notional architecture with the client
  • Define POC use cases and architectural experiments to refine the architecture

Delivers documented Solution Architecture, run-time cost including cloud consumption estimate, and use cases to move to a POC.​

Implementation

Proof of Concept (POC)
  • Select one or several ADP or client provided use cases
  • Short time-boxed implementation
  • Focus on key use cases, technical, or business risks
  • Implementation Planning
  • Refine architecture based on POC and experiment results

Delivers source code, updated design documentation, updated consumption estimate, and a groomed backlog of Sprint 0/Sprint 1 User Stories for a reference architecture or full implementation.​

Reference Architecture
  • Iterate on ADP and POC output until a final version of the refined architecture is reached
  • Architecture and Development of key use case(s) using Agile process
  • UX/UI Design
  • BA to build sprint backlog, ER, sequence, and other design documents
  • Automated QA for Unit, UI and Integration testing

Delivers source code and documentation for a scalable Reference Architecture pattern that can be re-applied to multiple full implement projects.​

Enterprise Development
  • ADP/POC, Reference Arch as input
  • UX/UI Design
  • Dev and QA using Agile process
  • BA grooms sprint backlog, and design documentation
  • Automated QA for Unit, UI and Integration testing
  • PM support in scheduling, resourcing, budget and risk tracking

Delivers an enterprise level development and implementation, including documentation and source code. Options include UX/UI, DevOps CI/CD, and automated QA.​
